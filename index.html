Classes Involved

REST Resource

Ws_VGP_RestResource

The Salesforce REST entry point. Exposes /api/v1/vgp/* to external systems and delegates all HTTP methods to the router.

  • @RestResource urlMapping /api/v1/vgp/*
  • doGet delegates to WebServiceRouter.route
Configuration

WebServiceRouter

The central traffic controller. Orchestrates the entire request from parsing to handler resolution to response writing.

  • route: builds ApiRequest, resolves handler, writes response
  • writeResponse: status code, Content-Type, JSON
  • Catches ApiNotFoundException, ApiBusinessException, ApiValidationException
DTO

ApiRequest

Wraps the raw Salesforce RestContext into a structured, easy-to-use request object with parsed URL segments and a unique request ID.

  • fromRestContext: wraps RestContext into ApiRequest
  • parseUrlSegments: apiVersion, resourceType, subResource
  • Instantiates QueryOptions.fromRequest
Configuration

QueryOptions

Parses query string parameters into structured pagination, sorting, and filtering options. Non-reserved params become filter entries.

  • fromRequest: fields, orderBy, orderDir, pageSize, pageNumber
  • calculateOffset: (pageNumber - 1) x pageSize
  • extractFilters: non-reserved params become filters
Configuration

HandlerRegistry

A metadata-driven lookup that finds the right Apex handler class for a given HTTP method + resource type. Uses Custom Metadata for configuration.

  • Queries API_Handler_Config__mdt
  • Type.forName to instantiate handler
  • Throws ApiNotFoundException / ApiBusinessException
Handler

Ws_VGP_GetVGPHandler

Handles all GET requests for the VGP resource. Routes to the correct service method based on the sub-resource in the URL.

  • extends BaseRequestHandler
  • switch on subResource: vgp-users
  • Throws ApiNotFoundException on invalid subResource
Handler - Abstract

BaseRequestHandler

A safety wrapper around every handler. Uses the Template Method pattern and catches all exceptions, converting them to proper API error responses.

  • Template Method: preProcess, execute, postProcess
  • Catches: 400, 404, 422, 500
Service

SiteUserService

Contains the business logic for site user operations. Coordinates selector calls, applies rules, and builds response DTOs.

  • getVGPUsers: countAll + selectAll
  • Builds SiteUserListDto + PaginationMeta
Selector

SiteUsersSelector

The data access layer for Site_User__c. Encapsulates all SOQL queries using fflib_QueryFactory for clean, testable database access.

  • countAll: SELECT COUNT() FROM Site_User__c
  • selectAll(limit, offset): fflib_QueryFactory
Exception

ApiException (abstract)

The base class for all custom API exceptions. Carries an HTTP status code and error code so the framework can automatically build the right error response.

  • statusCode Integer, errorCode String
Exception - 404

ApiNotFoundException

Thrown when a requested resource, handler, or sub-resource does not exist. Results in an HTTP 404 response.

  • errorCode NOT_FOUND, resourceType, resourceId
Exception - 422

ApiBusinessException

Thrown when a business rule is violated (e.g. duplicate email, missing handler class). Results in an HTTP 422 response.

  • errorCode BUSINESS_RULE_VIOLATION, businessRule

Complete Class Reference

REST Resource

Ws_VGP_RestResource

The Salesforce REST entry point. Exposes /api/v1/vgp/* to external systems and delegates all HTTP methods to the router.

  • @RestResource urlMapping /api/v1/vgp/*
  • doGet, doPost, doPatch, doDelete
  • All methods delegate to WebServiceRouter.route
Configuration

WebServiceRouter

The central traffic controller. Orchestrates the full lifecycle: build request, find handler, invoke it, write JSON response.

  • route: orchestrates full request lifecycle
  • writeResponse: serialises ApiResponse to RestContext
  • Catches ApiNotFoundException, ApiBusinessException, ApiValidationException
Configuration

HandlerRegistry

A metadata-driven lookup. Queries Custom Metadata to find which Apex class handles a given HTTP method + resource combination. Caches results.

  • getHandler(httpMethod, resourceType)
  • Queries API_Handler_Config__mdt custom metadata
  • Type.forName dynamic instantiation + cache
  • Throws ApiNotFoundException / ApiBusinessException
DTO

ApiRequest

Wraps the raw Salesforce RestContext into a clean, structured request object with parsed URL segments, headers, query params, and a unique ID.

  • fromRestContext: factory method
  • parseUrlSegments: version, resource, subResource
  • getBody(Type): delegates to RequestBodyParser
  • generateRequestId: 32-char hex
Configuration

QueryOptions

Parses query string parameters into structured pagination, sorting, and filtering options. Anything not in the reserved set becomes a filter.

  • fields, filters, orderBy, orderDir, pageSize, pageNumber, offset
  • extractFilters: non-reserved params become filters
  • Throws ApiValidationException for invalid page_size
Configuration

RequestBodyParser

Reads the JSON request body, deserialises it into a typed DTO, and auto-calls validate(). Centralises all body parsing logic.

  • parse(Type): reads RestContext body
  • JSON.deserialize to IRequestDto, auto-calls validate
  • Throws ApiValidationException on empty or invalid body
Interface

IRequestHandler

The contract that all request handlers must implement. Ensures every handler has a handle() method.

  • handle(ApiRequest) returns ApiResponse
Handler - Abstract

BaseRequestHandler

A safety wrapper around every handler using the Template Method pattern. Catches all exceptions and always logs timing.

  • Template Method: preProcess, execute, postProcess
  • Catches: 400, 404, 422, 500
  • ApiLogger in finally block
Handler

Ws_VGP_GetVGPHandler

Handles all GET requests for VGP. Uses a switch to route each sub-resource to the correct service method.

  • vgp-users → SiteUserService.getVGPUsers
  • get-cda → not implemented
  • Throws ApiNotFoundException on unknown subResource
Handler

Ws_VGP_PostVGPHandler

Handles all POST requests for VGP. Routes to service methods based on the sub-resource.

  • create-siteuser → SiteUserService.createSiteUser
  • Throws ApiNotFoundException on unknown subResource
Interface

ISiteUserService

The contract for the site user service. Allows the service to be swapped out or mocked in tests via the Application factory.

  • getVGPUsers(ApiRequest): ApiResponse
  • createSiteUser(ApiRequest): ApiResponse
Service

SiteUserService

The business logic layer. Coordinates selectors, applies rules (duplicate checks, password hashing), and builds response DTOs.

  • getVGPUsers: countAll + selectAll, paginated list
  • createSiteUser: validate, dup check, gen key, hash pw, insert
  • Throws ApiBusinessException DUPLICATE_EMAIL
Interface

ISiteUsersSelector

The contract for the selector. Defines what queries are available. Can be mocked in unit tests.

  • selectSiteUsersByEmail, selectAll, selectActiveUser
  • countAll, countAllActiveUsers
Selector

SiteUsersSelector

The data access layer for Site_User__c. Encapsulates all SOQL using fflib_QueryFactory for dynamic, testable queries.

  • extends fflib_SObjectSelector
  • Site_User__c: Id, Email__c, Priest__c
  • fflib_QueryFactory for dynamic SOQL
DTO

ApiResponse

The standardised response envelope. Every API response has the same structure: success flag, status code, data, errors, and pagination.

  • success, statusCode, requestId
  • data (IResponseDto), errors (ApiError list), pagination
DTO

ApiError

Represents a single error in the response. Can include a code, message, and optionally which field caused the error.

  • code, message, field
Interface

IResponseDto / IRequestDto

Marker interfaces. IResponseDto tags all response DTOs. IRequestDto requires a validate() method for input validation.

  • IResponseDto: marker interface
  • IRequestDto: validate() method
Configuration

ApiResponseBuilder

Factory for building standardised API responses. Provides methods for success (ok, okList) and error (notFound, forbidden, internalError) responses.

  • ok: 200 single, okList: 200 paginated
  • error, notFound 404, forbidden 403, internalError 500
Configuration

PaginationMeta

Computes pagination metadata from the total record count and page options. Included in list responses so consumers can navigate pages.

  • build(totalRecords, opts)
  • totalRecords, pageSize, pageNumber, totalPages, hasNext, hasPrev
Response DTO

SiteUserListDto

Transforms a list of Site_User__c records into a clean API format. Each record becomes a SiteUserItem with only the fields the consumer needs.

  • implements IResponseDto
  • SiteUserItem[]: sf_id, email, priest_sf_id
Response DTO

CreatedSiteUserDto

Transforms a newly created Site_User__c into the API response format, confirming the creation was successful.

  • implements IResponseDto
  • sf_id, firstName, lastName, email, priest_sf_id
Request DTO

CreateSiteUserRequestDto

Represents the expected JSON body for creating a user. Contains validation rules that check all required fields are present.

  • implements IRequestDto
  • firstName, lastName, email, password, priestId
  • Throws ApiValidationException on missing fields
Exception - Abstract

ApiException

The base class for all custom API exceptions. Carries an HTTP status code and error code so the framework knows how to respond.

  • extends Exception
  • statusCode Integer, errorCode String
Exception - 400

Thrown when input data is invalid. Carries a list of specific error messages so the consumer knows exactly what to fix.

Adding a New Endpoint - Step by Step

Follow these steps every time you need to add a new API endpoint. The architecture is already built, you only need to create the endpoint-specific pieces.

Step 0 Decide: New Resource or New Sub-Resource?

New sub-resource on existing resource

e.g. adding GET /vgp/active-users to the existing VGP resource

Skip to Step 3. Just add a new when case to the existing handler.

Completely new resource

e.g. adding GET /api/v1/orders/*

Start from Step 1. You need a new REST Resource class, handler, CMDT records, and everything below.

Step 1 Create the REST Resource class

Only needed for a completely new resource group. Skip if adding to an existing one (e.g. VGP already has one).

CREATE → WebServices/{Resource}/Ws_{Resource}_RestResource.cls
@RestResource(urlMapping='/api/v1/{resource}/*')
global with sharing class Ws_{Resource}_RestResource {
    @HttpGet
    global static void doGet() { WebServiceRouter.route(); }

    @HttpPost
    global static void doPost() { WebServiceRouter.route(); }
}

Step 2 Create Custom Metadata record

Register your handler in API_Handler_Config__mdt so the HandlerRegistry can find it. One record per HTTP method + resource combination.

CREATE → API_Handler_Config__mdt record
HTTP_Method__cGET (or POST, PATCH, DELETE) Resource_Type__cyour-resource (lowercase, matches URL segment) Handler_Class__cWs_{Resource}_Get{Resource}Handler (full class name) Is_Active__ctrue

Step 3 Create the Handler class

One handler per HTTP method per resource. Contains a switch statement that routes sub-resources to service methods. If adding a sub-resource to an existing handler, just add a new when case.

CREATE → WebServices/{Resource}/Handlers/Ws_{Resource}_{Method}Handler.cls
public with sharing class Ws_{Resource}_Get{Resource}Handler
    extends BaseRequestHandler {

    protected override ApiResponse execute(ApiRequest request) {
        switch on request.subResource {
            when 'your-sub-resource' {
                return YourService.service().yourMethod(request);
            }
            when else {
                throw new ApiNotFoundException(
                    'Invalid subresource', 'resource', request.subResource
                );
            }
        }
    }
}

Step 4 Create or update the Service

The service contains all business logic. Add your method to the interface and the implementation. Register it in Application.cls if it's a new service.

MODIFY → IYourService.cls (add method signature)
ApiResponse yourMethod(ApiRequest request);
MODIFY → YourService.cls (implement the method)
public ApiResponse yourMethod(ApiRequest request) {
    QueryOptions opts = request.options;
    // For POST: parse body
    // YourRequestDto dto = (YourRequestDto) request.getBody(YourRequestDto.class);

    // Call selectors
    // Apply business rules
    // Build response DTO
    return ApiResponseBuilder.ok(responseDto, request.requestId);
}
MODIFY → Application.cls (only if new service - register binding)
IYourService.class => YourService.class

Step 5 Create or update the Selector

Add any new SOQL queries your service needs. Register in Application.cls if it's a new selector.

MODIFY → IYourSelector.cls (add method signature)
List<Your_SObject__c> selectByCondition(params);
MODIFY → YourSelector.cls (implement query)
public List<Your_SObject__c> selectByCondition(params) {
    return Database.query(
        newQueryFactory()
            .setCondition('Field__c = :param')
            .toSOQL()
    );
}

Step 6 Create DTOs

Create request DTOs (for POST/PATCH) and response DTOs to shape the API input/output.

CREATE → WebServices/{Resource}/RequestDTO/YourRequestDto.cls (POST/PATCH only)
public class YourRequestDto implements IRequestDto {
    public String field1;
    public String field2;

    public void validate() {
        List<String> errors = new List<String>();
        if (String.isBlank(field1)) errors.add('field1 is required');
        if (!errors.isEmpty())
            throw new ApiValidationException('Validation failed', errors);
    }
}
CREATE → WebServices/{Resource}/ResponseDTO/YourResponseDto.cls
public class YourResponseDto implements IResponseDto {
    public String sf_id;
    public String fieldName;

    public YourResponseDto(Your_SObject__c record) {
        this.sf_id = record.Id;
        this.fieldName = record.Field__c;
    }
}
